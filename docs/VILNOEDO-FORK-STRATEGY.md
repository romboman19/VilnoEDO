# VilnoEDO — fork strategy

## Мета

Побудувати українсько-орієнтований self-hosted продукт електронного документообігу на базі Documenso, але з окремою продуктовою стратегією під український ринок.

## Що залишаємо від upstream як базу

- загальну модель документа
- маршрути підписання
- шаблони
- кабінет користувача
- керування підписантами
- історію дій
- storage / auth / admin foundation

## Що стає українською зоною переробки

### 1. Локалізація
- українська мова як основна
- термінологія під місцевий EDO-контекст
- адаптація текстів, статусів, назв ролей і сценаріїв

### 2. Підпис
- інтеграція з `VilnoCheck-SignService`
- КЕП / УЕП
- токени
- файлові ключі
- хмарні сценарії підпису
- відображення метаданих сертифіката і типу підпису

### 3. Бізнес-сценарії
- договір
- акт
- рахунок
- первинні документи
- внутрішні погодження

## Архітектурний принцип

Навіть у форку криптографічну частину не змішуємо з головним застосунком.

### VilnoEDO App
Відповідає за документний workflow.

### VilnoCheck-SignService
Відповідає за signing core.

## Тип форка

Поточне рішення: **controlled fork**.

Це означає:
- ми стартуємо з кодової бази Documenso
- певний час ще дивимося на корисні зміни upstream
- але продуктова логіка, локалізація і signing stack вже йдуть у власному напрямі

## Не робити на старті

- не намагатися одразу переписати весь продукт
- не змішувати signing core в UI/route/business logic випадковими вставками
- не ламати всі апстрімні модулі без карти змін

## Найближчі deliverables

1. Брендування під VilnoEDO
2. Карта модулів Documenso, які зачіпає українська адаптація
3. Контракт інтеграції з VilnoCheck-SignService
4. POC одного реального сценарію підпису
5. Backlog першого українського релізу
