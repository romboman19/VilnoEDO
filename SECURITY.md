# Політика безпеки / Security Policy

VilnoEDO працює з юридично значущими документами, тому безпека — пріоритет проєкту.

## Як повідомити про вразливість

Повідомляйте про вразливості приватно. Не відкривайте публічні issue, discussion чи pull request для security-звітів.

1. **GitHub Security Advisories (бажано)** — [форма приватного звіту](https://github.com/romboman19/VilnoEDO/security/advisories/new).
2. **Email** — [romboman19@gmail.com](mailto:romboman19@gmail.com).

Вкажіть версію, опис проблеми, кроки відтворення і потенційний вплив.

## Обсяг

Політика стосується коду застосунку VilnoEDO в цьому репозиторії. Питання розгортання, мережевої інфраструктури та конфігурації (TLS, заголовки, rate limiting, DoS, DNS) — відповідальність оператора інсталяції.

Вразливості, успадковані від upstream-проєкту [Documenso](https://github.com/documenso/documenso), варто також повідомляти в upstream.

## Підтримувані версії

Виправлення безпеки застосовуються до останнього релізу. Використовуйте найновішу версію.
