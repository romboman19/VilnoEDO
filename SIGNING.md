# Сертифікат запечатування (`.p12`) — технічна печатка, не КЕП

> **Важливо.** `.p12` — це **технічний** інстанс-сертифікат Documenso для запечатування
> звичайних (SES) документів. Він **не є** українським механізмом юридично значущого
> підпису і **не бере участі** в рішенні «документ підписано» для українського флоу.

## Український юридичний флоу (основний)

Юридично значущий підпис у VilnoEDO — це **КЕП / УЕП / електронна печатка**, які:

- створюються **на стороні клієнта** через ІІТ (web extension / web library); приватний ключ на сервер не виходить;
- сервер приймає готовий detached-артефакт, перевіряє прив'язки (сесія, отримувач, конверт, хеш документа) і формує evidence package;
- повна криптографічна перевірка (ДСТУ-4145, ланцюг, відкликання) виконується окремим ІІТ-механізмом (див. [README](README.md)).

`.p12` цей ланцюг **не замінює** і для нього не потрібен.

## Технічна печатка `.p12` (опційно, для звичайних SES-документів)

Використовується лише для upstream-флоу Documenso (SES). Без неї застосунок стартує; на
UA КЕП/УЕП флоу її відсутність не впливає.

```bash
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 730 -nodes
openssl pkcs12 -export -out cert.p12 -inkey key.pem -in cert.pem
```

Шлях — `NEXT_PRIVATE_SIGNING_LOCAL_FILE_PATH` (у проді монтується як `/opt/documenso/cert.p12`,
див. [deploy/compose.yml](deploy/compose.yml)), пароль — `NEXT_PRIVATE_SIGNING_PASSPHRASE`.

## TLS ≠ підпис

TLS-сертифікат (Let's Encrypt тощо) потрібен лише для захищеного HTTPS-доступу до вебзастосунку.
Це **не** сертифікат електронного підпису чи печатки документа і до юридичного підписання
відношення не має.
